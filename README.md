# Предметная область

Интеллектуальное управление группой лифтов, с целью сокращения времени
ожидания, снижения энергопотребления, увеличения средней загруженности.

# Доступные пользователям интерфейсы

Веб-страница для администратора для мониторинга метрик, отчётов

REST API для интеграции с внешними сервисами (камерами, датчиками в
лифтах)

По сути сам пользователь с системой никак напрямую не взаимодействует.
По итогу разработанная система должна подменить систему управления
лифтами.

# Взаимодействие с внешними системами

Интеграция с датчиками загрузки лифтов, кнопками на этажах, камерами на
этажах для определения количества людей перед лифтом и на этажах

Чтение расписания занятий? (потенциально можно, сложно)

# Внутреннее разделение на подсистемы и модули, и интерфейсы взаимодействия между ними

-   Подсистема сбора данных:

    -   Датчики лифтов

    -   Кнопки лифтов

    -   Камеры

-   Подсистема анализа данных.

-   Подсистема прогноза.

-   Подсистема распределения (ИИ решает как распределить лифты в
    следующий момент)

-   Подсистема управления (интерфейс лифта)

-   Подсистема мониторинга и аналитики (собирает информацию по
    фактическим результатам работы)

# Используемые модели и алгоритмы машинного обучения и искусственного интеллекта

LSTM / TFT --- для прогнозирования временных рядов пассажиропотока.

XGBoost / LightGBM --- как бэйзлайн и для гибридных моделей.

Proximal Policy Optimization (PPO) / Deep Q-Network--- для оптимизации
стратегии распределения вызовов.

# Используемые методы обучения и требования к наборам обучающих данных

Обучение на исторических данных вызовов лифтов, сегментированных по
времени суток, дням недели, событиям (пары, обеды и т.д.).

Требования к данным:

Минимум 3 месяца истории вызовов. (генерируется имитационной моделью)

Метки: этаж вызова, направление, время, загрузка кабины.

Дополнительно: расписание занятий, план здания, данные с датчиков.

Использование онлайн-дообучения и transfer learning при переходе на
новое здание.

# Используемые процедуры валидации и тестирования системы

Имитационное моделирование

Сравнение с базовыми алгоритмами управления лифтами

Метрики: AWT, MWT, коэффициент загрузки, энергопотребление, % пустых
поездок.

A/B-тестирование в симуляции перед внедрением.

# Варианты развёртывания системы

On-premise -- локальный сервер в здании, система в контейнере Docker

Гибридное облако --- обучение в облаке, inference на edge-устройствах.

Полностью облачный -- Docker / Kubernetes, если нужно масштабирование

# Процедуры подготовки к развёртыванию системы

Аудит совместимости с лифтовым оборудованием.

Настройка датчиков и интеграционных шлюзов.

Калибровка модели под конкретное здание (fine-tuning).

Проведение пилотного запуска в симуляции → на одном лифте → на всей
группе.
